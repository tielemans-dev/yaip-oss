# CI Pnpm Version Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix the failing GitHub Actions bootstrap step by declaring a single pnpm version in repository metadata and guarding it with a local test.

**Architecture:** Keep workflow behavior unchanged except for how `pnpm` is resolved. Store the pnpm version in `package.json` via `packageManager`, let all workflows inherit that value, and add a focused test that detects accidental removal.

**Tech Stack:** GitHub Actions, pnpm, Node.js, Vitest, TypeScript

---

### Task 1: Add a failing repository metadata test

**Files:**
- Create: `scripts/__tests__/package-manager.test.ts`
- Modify: `package.json`

**Step 1: Write the failing test**

Create a Vitest test that reads `package.json` and asserts:

- `packageManager` exists
- it starts with `pnpm@`

**Step 2: Run test to verify it fails**

Run: `pnpm exec vitest run scripts/__tests__/package-manager.test.ts`

Expected: FAIL because `package.json` does not currently declare `packageManager`.

**Step 3: Write minimal implementation**

Add `"packageManager": "pnpm@10.28.2"` to `package.json`.

**Step 4: Run test to verify it passes**

Run: `pnpm exec vitest run scripts/__tests__/package-manager.test.ts`

Expected: PASS
